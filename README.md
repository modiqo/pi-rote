# pi-rote

Thin pi extension that makes rote the default workflow harness inside pi while keeping rote itself as the execution surface.

## What it does

- keeps native pi file tools for local code and file work
- teaches the model to use raw `rote ...` commands through `bash` for workflow, adapter, and API tasks
- auto-installs the core `rote` skill from the installed `rote` CLI into `${PI_CODING_AGENT_DIR:-~/.pi/agent}/cache/pi-rote/skills/rote` when the generated skill is missing or stale for the current `rote --version`
- bundles the standalone `rote-adapter` skill as part of the package
- exposes both skills to pi through extension-driven resource discovery
- detects likely non-rote workflow/API commands and appends rote hints instead of blocking them
- mechanically probes `rote explore --json` and `rote adapter catalog search --json` to see whether installed or catalogued rote support exists
- keeps a chat-local active rote context hint so the model remembers the current workspace, path, and initialized adapter sessions
- nudges the model into rote's lifecycle: **search → execute → crystallize → reuse**

## Requirements

- `pi` installed
- `rote` installed and available on `PATH`

## Install locally

```bash
pi install /absolute/path/to/pi-rote
```

Then start a fresh pi session in any project. On startup, `pi-rote` will try to materialize the core `rote` skill automatically if `rote` is available on `PATH`.

## Quick test without installing

```bash
cd /absolute/path/to/pi-rote
pi -e ./extensions/rote.ts
```

This direct extension mode should still expose the bundled `rote-adapter` skill and will bootstrap the generated core `rote` skill into the pi cache when possible.

## Behavior contract

- raw `rote ...` commands are allowed
- local development commands such as `cargo test` and `git status` are allowed
- likely workflow/API commands such as `gh issue ...` or `curl https://api.github.com/...` receive a rote hint when rote is available and the extension can find installed or catalogued rote support
- if `rote` is missing, the extension only appends guidance and does not attempt mechanical rote support probes
- when a rote workflow has already started in the current chat, the prompt reminds the model which workspace/path/session context to re-enter before issuing workspace-bound rote commands

## Generated skill cache

The extension materializes the core `rote` skill into:

```bash
${PI_CODING_AGENT_DIR:-~/.pi/agent}/cache/pi-rote/skills/rote
```

Files created there:
- `SKILL.md` — generated by `rote install skill --provider cursor --path ...`
- `.pi-rote-rote-version` — the `rote --version` value used to decide whether the cache is stale

## Manual smoke checks

1. Start pi with the extension:

   ```bash
   pi -e ./extensions/rote.ts
   ```

2. Ask for a rote-worthy task:

   ```text
   list my GitHub issues and save the workflow if it looks reusable
   ```

   Expected behavior:
   - the model starts with `rote flow search ...`
   - if no flow exists, it moves into rote execution instead of calling `gh` directly
   - after a successful reusable result, it treats crystallization as the next step

3. After the first rote workflow step succeeds, ask a follow-up that depends on the same workspace:

   ```text
   now inspect the previous rote result and continue in the same workspace
   ```

   Expected behavior:
   - the prompt has refreshed the active rote context hint
   - the model re-enters the remembered workspace directory before running workspace-bound rote commands

4. Ask for a normal coding task:

   ```text
   run cargo test and summarize failures
   ```

   Expected behavior:
   - the model uses normal `bash`
   - the hint layer does not interfere

5. Confirm a direct workflow/API bypass receives a rote hint when rote is available:

   ```text
   use gh issue list to inspect my GitHub issues
   ```

   Expected behavior:
   - the command still runs
   - the tool result includes a `rote hint:` block
   - the hint mentions installed rote support or an adapter catalog candidate and suggests concrete rote commands to try next
